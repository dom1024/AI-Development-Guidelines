# Architecture Template

Use this template to define project architecture before any Cursor execution, incorporating Claude-inspired multi-phase task planning.

## Sections

### 1. System Overview
- Project goal
- Scope
- Key constraints
- Critical tasks marked explicitly
- Human review required for high-impact changes

### 2. Modules
- Module names
- Responsibilities
- Dependencies
- Phase assignment (Plan / Execute / Verify)

### 3. Data Flow
- Inputs
- Outputs
- Data stores
- Event triggers
- Phase-specific processing notes

### 4. Interfaces
- API endpoints
- Function signatures
- Expected outputs
- Validation requirements

### 5. Decisions
- Tech stack choices
- Trade-offs
- Known limitations
- Decision rationale logged for each phase

### 6. Phase Plan
- Phase 1: … (Plan)
- Phase 2: … (Execute)
- Phase 3: … (Verify)
- Next steps and dependencies clearly noted

### 7. Notes
- Known issues
- Future improvements
- Phase tracking instructions
